Well-formed FXML from design tools such as Scene Builder can be supplied directly to the FXDeployment code.

For an overview of FXML see here and for a full specification of FXML see here.
As a convenience, the code also allows FXML lacking the required header and import information to be supplied. This following is added:
  1. Define the language and encoding:
    <?xml version="1.0" encoding="UTF-8"?>
  2. The standard imports:
    <?import java.lang.*?>
    <?import java.util.*?>
    <?import javafx.scene.effect.*?>
    <?import javafx.geometry.Pos.*?>
    <?import javafx.scene.text.*?>
    <?import javafx.scene.*?>
    <?import javafx.scene.control.*?>
    <?import javafx.scene.shape.*?>
    <?import javafx.scene.layout.*?>
    <?import javafx.collections.*?>
In addition, namespaces will be added to the root element e.g.
<AnchorPane ... xmlns:fx = "http://javafx.com/fxml/1" xmlns = "http://javafx.com/javafx/8" >
...
</AnchorPane>

The imports in 2 above can be modified using fxdeploy$common methods:
  • getHeader() returns the header string
  • setHeader(newString) replaces it so additional imports can be appended
