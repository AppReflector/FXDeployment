What does the FXDeployment code do?

The FXDeployment code allows GUIs described using the FXML markup language to be embedded into a web page. FXML provides access to a wide range of GUI control elements (buttons, checkboxes, menus, date selectors etc.) together with containers for them including border panes, stack panes, grid panes and anchor panes. See here for details.
The FXWebApp class included in the FXDeployment package automatically generates the required JavaFX GUI hierarchy and provides access to the GUI elements from the browser using Javascript.

Do I need to work in Java?

No.
The GUI design is specified with the FXML markup language and will be automatically processed. All a user need do is add an element to the HTML for their web page to contain the GUI. The declaration of that element can specify the FXML file as an attribute - or the FXML can be in-lined as the content of the HTML element. Full details are provided in "Getting Started".

How it works

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
