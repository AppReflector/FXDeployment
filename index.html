What does the FXDeployment code do?

The FXDeployment code allows GUIs described using the FXML markup language to be embedded into a web page. FXML provides access to a wide range of GUI control elements (buttons, checkboxes, menus, date selectors etc.) together with containers for them including border panes, stack panes, grid panes and anchor panes. See here for details.
The FXWebApp class included in the FXDeployment package automatically generates the required JavaFX GUI hierarchy and provides access to the GUI elements from the browser using Javascript.

Do I need to work in Java?

No. The FXWebApp class is called from Javascript and allows all end-user work to be done with HTML, FXML, Javascript and, if required, CSS.
All a user need do is add an element to the HTML for their web page to contain the GUI. The declaration of that element can specify an FXML file as an attribute - or the FXML can be in-lined within the HTML element.
The init function in the accompanying Javascript file (fxdeploy-common.js) should be called from the HTML document onload method. This will invoke the required Java code which will:
  1. Embed the GUI in the web page
  2. Embed an application in the HTML document that provides access to the Java GUI elements from Javascript (and exposes some convenience Java methods to Javascript programmers
  3. Attaches Javascript callbacks to the GUI elements that can be programmed to respond to user-generated events such as button clicks, combobox selection changes etc.
Full details are provided in the "Getting Started" pages.

Where it works

The code is presently targeted at browsers. Only those that support Java/JavaFX can use this code. These currently include:
  • Windows - Internet Explorer, Firefox
  • Mac OS - Safari, Firefox
Recent releases of Chrome, and other Chromium-based browsers, have removed support for NPAPI and can not display JavaFX GUIs.
