Step 1: Adding the Javascript

The FXWebApp code uses two Javascript files.
  1. dtjava.js which is part of the Java Deployment Toolkit
  2. fxdeploy-common.js which is part of the FXDeployment code
Use of fxdeploy-common.js is optional: you can directly call the functions supplied in dtjava.js, but fxdeploy-common.js makes this easier.
In the description that follows, it is assumed that fxdeploy-common.js is being used. Add the <script> declarations to the <head> element of your HTML file:
<script type="text/javascript" src="http://java.com/js/dtjava">
<script type="text/javascript" src="pathto/fxdeploy-common.js">
Note that you need to set the path to locate the fxdeploy.common.js file on your site.
Once loaded, a variable named fxdeploy$common will be available defining an init function that should be called as described below after loading the HTML document.

Step 2: Adding the required HTML

fxdeploy-common.js needs to know where in your HTML document it should embed the GUI. Create an HTML element with an attribute of:
  • fxml-placeholder if you are using an FXML file
  • fxml-content if you are specifying the FXML inline within the element
Both element classes should set the following attributes for use by fxdeploy-common.js:
  • data-appname - provides a name for the embedded Java application. The application may be accessed, once loaded, by calling to document.getElementById(data-appname);
  • data-callbacks - a string providing the name for the variable containing the JavaScript callbacks for the JavaFX node. If data-callbacks is not defined, a default value of fxml$callbacks will be assigned.
  • data-jnlp provides the fully qualified path and file name for the JNLP file for this project.
For a fxml-placeholder, the "src" attribute should also be set to provided the fully qualified path to the FXML file.

Step 3: Invoking the Javascript on HTML document loading

Call the fxdeploy$common.init() function as part of the document onload sequence e.g.
<body onload="fxdeploy$common.init()">
This will:
  • Replace the content of the fxml-placeholder or fxml-content element with the GUI
  • Embed the application (named as specified by data-name) into the document
  • Attach Java listeners to the GUI controls that will forward events to Javascript callbacks available in the variable specified by data-callbacks.
The application will be an instance of the FXWebApp class. For details of the methods available to Javascript programmers by calling into this app, see the here.
Details of how to write the Javascript callbacks are given in Attaching Callbacks.
